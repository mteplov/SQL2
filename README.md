### Задание 2


Разработайте план для выполнения горизонтального и вертикального шаринга базы данных. База данных состоит из трёх таблиц: 

- пользователи, 
- книги, 
- магазины (столбцы произвольно). 

Опишите принципы построения системы и их разграничение или разбивку между базами данных.

```
Горизонтальный шардинг (Horizontal Sharding) – это разделение данных по строкам таблиц.
Пример: таблицу пользователи можно разбить по регионам или по ID: пользователи с ID 1-1000 на сервере A, 1001-2000 на сервере B.
Цель: распределить нагрузку на чтение/запись, ускорить доступ.
```
```
Вертикальный шардинг (Vertical Sharding) – это разделение данных по столбцам таблиц.
Пример: таблицу книги можно разделить на «основные данные» (название, автор) на одном сервере и «детали» (описание, отзывы, рейтинг) на другом сервере.
Цель: уменьшить объём данных на одном сервере и оптимизировать часто используемые столбцы.
```
## 3. Принципы построения системы

### Горизонтальный шардинг
- **Таблица `пользователи`**:
  - Сервер 1: ID 1–1000  
  - Сервер 2: ID 1001–2000  
  - Сервер 3: ID 2001+  

- **Таблица `магазины`**:
  - Разделение по географии (Москва, СПБ, регионы)  

- **Таблица `книги`**:
  - Разделение по жанрам или алфавиту названия  

### Вертикальный шардинг
- **Таблица `книги`**:
  - Сервер A: основные данные (ID, название, автор)  
  - Сервер B: дополнительные данные (рейтинг, описание, отзывы)  

- **Таблица `пользователи`**:
  - Сервер A: основные данные (ID, имя, email)  
  - Сервер B: профиль и настройки (адрес, предпочтения)

## 4. Режимы работы серверов

| Сервер      | Роль                                  | Режим работы                     |
|------------|--------------------------------------|----------------------------------|
| DB-User-1  | Часть пользователей                   | Активный, чтение/запись          |
| DB-User-2  | Остальные пользователи                | Активный, чтение/запись          |
| DB-Shop-1  | Магазины по регионам                  | Активный, чтение/запись          |
| DB-Book-1  | Основные данные книг                  | Активный, чтение/запись          |
| DB-Book-2  | Дополнительные данные книг            | Активный, чтение/запись          |

> Примечание: рекомендуется настроить репликацию на резервные сервера для отказоустойчивости.

## Блок-схема шардинга
``` text 
             +----------------+
             |   Приложение   |
             +----------------+
                     |
     -----------------------------------
     |                |                |
+------------+   +------------+   +------------+
|  DB-User-1 |   |  DB-User-2 |   |  DB-Shop-1 |
| Горизонт.  |   | Горизонт.  |   | Горизонт.  |
| шардинг    |   | шардинг    |   | шардинг    |
+------------+   +------------+   +------------+
     |                |                |
     |                |                |
     +----------------+----------------+
                      |
             +----------------+
             |    DB-Book-1   |
             | Вертикальный   |
             | шардинг        |
             +----------------+
                      |
             +----------------+
             |    DB-Book-2   |
             | Вертикальный   |
             | шардинг        |
             +----------------+
```

- **Вертикальный шардинг:** таблица `книги` делится по столбцам.
